<objective>
Build the config persistence layer (JSON at ~/.claude-code-proxy/config.json), Keychain integration (keytar npm), and admin REST API endpoints. API keys are stored in macOS Keychain — never in config files or logs. The admin API exposes CRUD for providers and routes per D-05.

Purpose: Without config persistence, nothing survives a restart. Without Keychain, API keys are exposed. Without admin API, no runtime configuration. Together they make the proxy configurable from the UI (Phase 3) and CLI.
Output: ConfigService, KeychainService, admin API routes, test scaffolds
</objective>

  <name>Task 3: Wire admin API endpoints into Express server</name>

  <action>
    Create packages/proxy/src/routes/admin.ts implementing D-05 admin endpoints:

    ```typescript
    import { Router } from 'express';
    import { configService } from '../services/config';
    import { keychainService } from '../services/keychain';
    import { providerService } from '../services/provider';
    import { z } from 'zod';

    const router = Router();

    // GET /config — return current config (no keyId values exposed)
    router.get('/config', (req, res) => {
      const config = configService.load();
      res.json(config); // keyId only, not actual keys (D-14)
    });

    // PUT /config — save config, reload provider service
    router.put('/config', async (req, res) => {
      const config = req.body;
      configService.save(config);
      providerService.reload(config.providers, config.routes);
      res.json({ success: true });
    });

    // GET /providers — list all providers (masked keys)
    router.get('/providers', async (req, res) => {
      const providers = providerService.getProviders();
      // For each provider, mask the key for display
      const masked = await Promise.all(providers.map(async (p) => {
        const hasKey = await keychainService.hasKey(p.name);
        return {
          name: p.name,
          baseUrl: p.baseUrl,
          keyId: p.keyId,
          keyMask: hasKey ? '••••' : null, // AUTH-03: masked display
          models: p.models,
          enabled: p.enabled,
          priority: p.priority,
        };
      }));
      res.json(masked);
    });

    // POST /providers — add new provider, store key in Keychain
    router.post('/providers', async (req, res) => {
      const { name, baseUrl, keyId, models, apiKey, enabled, priority } = req.body;
      // Store actual key in Keychain (D-08, D-09)
      if (apiKey) {
        await keychainService.setKey(name, apiKey);
      }
      // Register provider (keyId stored in config, not actual key per D-14)
      providerService.registerProvider({ name, baseUrl, keyId, models, enabled, priority });
      res.json({ success: true, keyId });
    });

    // DELETE /providers/:id — remove provider and its Keychain entry
    router.delete('/providers/:id', async (req, res) => {
      const id = req.params.id;
      await keychainService.deleteKey(id);
      providerService.deleteProvider(id);
      res.json({ success: true });
    });

    // GET /routes — list model routes
    router.get('/routes', (req, res) => {
      const config = configService.load();
      res.json(config.routes);
    });

    // PUT /routes — update routes
    router.put('/routes', (req, res) => {
      const { routes } = req.body;
      const config = configService.load();
      config.routes = routes;
      configService.save(config);
      providerService.setRoutes(routes);
      res.json({ success: true });
    });

    export default router;
    ```

    - `app.use('/admin', adminRouter)` — per D-04 (admin endpoints on proxy server)

    Per D-05: Endpoints match exactly — GET/PUT /config, POST/GET/DELETE /providers, GET/PUT /routes.

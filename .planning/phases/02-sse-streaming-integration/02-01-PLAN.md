Create packages/proxy/src/adapters/openrouter.ts implementing ProviderAdapter (per D-15, D-17):

**providerType:** 'openrouter'

**timeouts:** { streaming: 120_000, nonStreaming: 30_000 } (per D-21)

**transformRequest():** OpenRouter supports native Anthropic `/v1/messages` API. Return body as-is with `stream: true` forced. No format conversion needed — this is a passthrough adapter.

**transformResponse():** OpenRouter returns native Anthropic SSE. Use eventsource-parser to parse the stream and pass through events, filtering out `[DONE]` terminal noise. Implementation:
1. Create parser with createParser from eventsource-parser
2. Read upstream response body as ReadableStream
3. For each parsed event: if data === '[DONE]', skip; otherwise yield `event: ${event.event || 'message'}\ndata: ${event.data}\n\n`
4. Handle stream reading with TextDecoder and buffer management (see RESEARCH.md SSE parser pattern, lines 373-404)

**validate():** Per D-23, D-24 — try `GET /v1/models` first (OpenRouter supports this). If successful, return `{ valid: true, models: data.data?.map(m => m.id) }`. If GET fails, return `{ valid: false, error: ... }`. Use AbortSignal.timeout(10_000) for the validation request. No POST fallback needed for OpenRouter — it supports GET /v1/models natively.

Import ProviderAdapter, AnthropicMessagesBody, TransformOptions, ValidationResult from './interface.js'. Import RouteResolution from '../types/index.js'. Import createParser from 'eventsource-parser'.

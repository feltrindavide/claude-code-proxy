  <action>
Create packages/proxy/src/services/sse-transformer.ts with:

**1. formatSSEEvent(eventType: string, data: Record<string, unknown>): string**
Returns `event: ${eventType}\ndata: ${JSON.stringify(data)}\n\n`

**2. STOP_REASON_MAP and mapStopReason(openaiReason: string | null): string**
Map: stop→end_turn, length→max_tokens, tool_calls→tool_use, content_filter→end_turn, default→end_turn

**3. ContentBlockManager class**
Tracks open content blocks and synthesizes missing start/stop events (addresses Pitfall #1 from RESEARCH.md):
- `nextIndex: number` — auto-incrementing index counter
- `textStarted: boolean` — whether text block has been opened
- `textIndex: number` — index of the text block
- `allocateIndex(): number` — returns next index and increments
- `ensureTextBlock(): string[]` — returns [content_block_start] if not already started
- `startTextBlock(): string` — creates content_block_start event for text
- `closeContentBlocks(): string[]` — returns [content_block_stop] for any open blocks

**4. SSEBuilder class**
Constructor: (messageId: string, model: string, inputTokens: number)
Methods:
- `message_start(): string` — emits message_start with id, model, role=assistant, content=[], stop_reason=null, usage={input_tokens, output_tokens: 1}
- `message_delta(stopReason: string, outputTokens: number): string` — emits message_delta with delta.stop_reason and usage
- `message_stop(): string` — emits message_stop
- `ensureTextBlock(): string[]` — delegates to ContentBlockManager, returns events array
- `emitTextDelta(content: string): string` — emits content_block_delta with text_delta
- `closeContentBlocks(): string[]` — delegates to ContentBlockManager, returns events array
- `emitTopLevelError(errorMessage: string): string` — emits error event with Anthropic error format (per D-26): `{type: 'error', error: {type: 'api_error', message: errorMessage}}`

**5. parseSSEStream(stream: ReadableStream<Uint8Array>): AsyncIterable<ParsedEvent>**
Uses eventsource-parser createParser. Read stream with TextDecoder, feed chunks to parser, yield parsed events. Handle remaining buffer on stream end. (See RESEARCH.md lines 373-404 for the exact pattern.)

**6. getUserFacingErrorMessage(error: unknown, timeoutMs?: number): string**
Sanitize errors for user display (per D-28):
- AbortError → "Provider request timed out after Ns."
- TypeError with 'fetch' → "Could not connect to provider."
- Error → sanitize by removing sk-* patterns, return message or "Provider request failed."
- Unknown → "Provider request failed unexpectedly."

Import createParser and ParsedEvent from 'eventsource-parser'.

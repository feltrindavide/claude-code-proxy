## Spacing Scale

Declared values (must be multiples of 4, from DESIGN.md spacing system):

| Token | Value | Usage |
|-------|-------|-------|
| xxs | 4px | Icon gaps, inline padding, status dot size |
| xs | 8px | Compact element spacing, form field gaps, label-to-input spacing, sidebar item padding |
| md | 16px | Default element spacing, card padding, section padding |
| lg | 24px | Form group spacing, layout gaps |
| xl | 32px | Sidebar width margin, major section breaks |
| 2xl | 48px | Page header spacing |
| 3xl | 64px | Page-level spacing (not used in app shell — marketing-only) |

Exceptions: 
- Touch targets minimum 44px height (WCAG AAA) for buttons and interactive elements
- Sidebar width: 240px (fixed, not from spacing scale)
- Status dot: 8px diameter (xs token)

**Source:** DESIGN.md spacing system + CONTEXT.md D-36 (status indicator)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f7f7f4` (canvas) | Main content background, page floor |
| Secondary (30%) | `#ffffff` (surface-card) | Sidebar, cards, form surfaces |
| Accent (10%) | `#f54e00` (Cursor Orange) | Primary CTAs only: "Save Provider", "Test Connection", "Start Proxy" buttons |
| Destructive | `#cf2d56` (semantic-error) | Delete provider confirmation, error states |

**Supporting colors:**
- Text primary: `#26251e` (ink)
- Text secondary: `#5a5852` (body)
- Text muted: `#807d72` (muted)
- Hairline border: `#e6e5e0` (hairline)
- Hairline strong: `#cfcdc4` (hairline-strong)
- Success: `#1f8a65` (semantic-success) — status running indicator, validation success

**Accent reserved for:**
1. Primary action buttons (Save, Test Connection, Start Proxy)
2. Active/selected sidebar navigation item indicator
3. Brand wordmark in app header

**Status indicator colors:**
- Running: `#1f8a65` (semantic-success) — green dot
- Stopped: `#807d72` (muted) — gray dot
- Error: `#cf2d56` (semantic-error) — red dot
- Loading: `#f54e00` (Cursor Orange) — animated pulsing dot

## Interaction Contracts

### Proxy Lifecycle
| Action | Trigger | Loading State | Success State | Error State |
|--------|---------|---------------|---------------|-------------|
| Start Proxy | Click "Start Proxy" button | Button disabled, spinner, "Starting proxy..." | Status dot → green, toast "Proxy started" | Status dot → red, error banner |
| Stop Proxy | Click "Stop Proxy" button | Button disabled, spinner, "Stopping proxy..." | Status dot → gray, toast "Proxy stopped" | Status dot → red, error banner |
| Health Poll | Every 5s (auto) | Status dot pulses orange during check | Status dot updates to current state | Status dot → red after 3 consecutive failures |

### Provider Management
| Action | Trigger | Validation | Success State | Error State |
|--------|---------|------------|---------------|-------------|
| Add Provider | Click "Add Provider" → fill form → "Save Provider" | Name required, base URL valid format, API key required | Toast "Provider saved", list updates | Inline field errors, form stays open |
| Test Connection | Click "Test Connection" on form | Base URL required, API key required | Button → green check, "Connection successful" | Button → red X, error message below |
| Delete Provider | Click trash icon → confirm dialog | Confirmation required | Provider removed from list, toast | Error banner if deletion fails |
| Toggle Provider | Click toggle switch | None | Toggle updates, provider enabled/disabled | Toggle reverts on failure |

### Navigation
| Action | Trigger | Behavior |
|--------|---------|----------|
| Switch page | Click sidebar nav item | Active item highlighted with orange left border, content updates |
| Auto-start | App launch | Proxy starts automatically if not running (CONTEXT.md D-32) |

## Page Specifications

### Status Page (Home)
- **Layout:** Single column, centered content area
- **Content:**
  - Status indicator (large dot + text label: "Running" / "Stopped" / "Error")
  - Status cards grid (2x2): Port, Version, Uptime, Provider Count
  - Recent errors section (if any) — last 5 errors with timestamp
- **Actions:** Start/Stop Proxy button (primary CTA)
- **Polling:** Health check every 5s

### Providers Page
- **Layout:** List view with "Add Provider" button (top-right)
- **Content:**
  - Provider list: name, type badge, enabled/disabled toggle, edit/delete actions
  - Empty state when no providers
- **Actions:** Add Provider (opens form), Edit (opens form pre-filled), Delete (confirmation dialog)
- **Form:** Modal or slide-over panel with all fields from CONTEXT.md D-42

### Model Mapping Page
- **Layout:** Form-based configuration
- **Content:**
  - Three tier rows: Opus, Sonnet, Haiku
  - Each row: tier label, provider dropdown, model identifier input
  - Save button (primary CTA)
- **Note:** Full model mapping UI scope — basic form for Phase 3, advanced features in Phase 4

### Settings Page
- **Layout:** Form sections
- **Content:**
  - Proxy port configuration (default: 3456)
  - Auto-start on launch toggle
  - Health check interval (default: 5s, read-only for Phase 3)
  - Export/Import configuration (placeholder for Phase 4)

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| WCAG AA contrast | All text meets 4.5:1 minimum against backgrounds |
| Keyboard navigation | All interactive elements focusable, visible focus ring (2px orange outline) |
| Screen reader | ARIA labels on status indicators, form fields, and navigation |
| Touch targets | Minimum 44px height for all buttons and interactive elements |
| Loading states | aria-busy on content areas during async operations |
| Error announcements | aria-live="polite" on error banners and toast notifications |

   - Detect shell from process.env.SHELL (zsh → .zshenv, bash → .bashrc, fish → ~/.config/fish/config.fish, fallback → .zshenv)
   - Write `export ANTHROPIC_BASE_URL="http://localhost:3456"` to shell profile

   - Default config structure: `{ providers: [], routes: [], version: "0.1.0" }`
   - Write with mode 0o600, skip if file already exists

   - If proxy is running (poll http://localhost:3456/health), fetch GET /admin/providers

   - If proxy not running, skip with warning message

   - Prompt user for backup file path (use readline for stdin input)
   - Read JSON file, validate it has providers[] and routes[] arrays
   - Merge with existing config (add new providers, update existing by name)

   - Support --import flag for non-interactive mode: `npm run setup -- --import /path/to/backup.json`

5. **Configure Keychain** (interactive):
   - For each provider in config, check if API key exists in Keychain (service: 'claude-code-proxy', account: provider.keyId)
   - If missing, prompt user for API key (use readline, hide input)
   - Store via keytar.setPassword('claude-code-proxy', provider.keyId, apiKey)
   - Support --non-interactive flag to skip Keychain prompts

   - Print: proxy health status (curl http://localhost:3456/health)
   - Print: config file location and existence
   - Print: number of configured providers
   - Print: Keychain status for each provider (key exists or missing)
   - Print: ANTHROPIC_BASE_URL value from process.env
   - Print: Node.js version, npm version

- `--non-interactive`: Skip all interactive prompts
- `--no-keychain`: Skip Keychain configuration
